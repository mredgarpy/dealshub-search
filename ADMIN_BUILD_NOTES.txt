CRITICAL IMPLEMENTATION DETAILS
================================================================================

1. AUTHENTICATION PATTERN
   All CRM API calls use query parameter token:
   
   fetch('/api/crm/ENDPOINT?token=' + encodeURIComponent(token))
   
   Token obtained from:
   - User input during login
   - localStorage.getItem('sh_crm_token')
   - Default: 'stylehub-admin-2026'

2. ENDPOINTS CALLED
   GET /api/crm/dashboard?token=X
   GET /api/crm/orders?status=FILTER&token=X
   GET /api/crm/customers?segment=FILTER&token=X
   GET /api/crm/returns?token=X
   GET /api/crm/reviews?status=FILTER&token=X
   GET /api/crm/tickets?status=FILTER&token=X
   GET /api/crm/tickets/stats?token=X
   GET /api/crm/analytics?period=PERIOD&token=X
   POST /api/crm/orders/:id/fulfill?token=X
   PUT /api/crm/reviews/:id?token=X
   POST /api/crm/settings?token=X
   GET /api/source-health (NO token needed - public endpoint)

3. NO BACKEND CHANGES REQUIRED
   The UI uses existing endpoints as-is.
   No modifications to server.js, crm-api.js, or tickets-api.js.

4. CHARTS
   All charts use pure CSS (no Chart.js or libraries).
   Bar charts implemented with flexbox and height percentages.

5. STORAGE
   Token stored in localStorage as 'sh_crm_token'.
   No other data persistence.

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "Loading..." stays forever
FIX:
  1. Check browser console (F12) for errors
  2. Verify API is running on correct port
  3. Check token is correct in input
  4. Check network tab (F12 > Network) for failed requests
  5. Verify /api/crm/dashboard endpoint exists and responds

ISSUE: Pages show "Failed to load"
FIX:
  1. Check API endpoint exists
  2. Verify token parameter is being sent
  3. Check API response format matches expected shape
  4. Check server logs for errors

ISSUE: Buttons don't work (Fulfill, Approve, Save)
FIX:
  1. Check browser console for JavaScript errors
  2. Verify POST/PUT endpoints exist
  3. Check request body format
  4. Verify API handles the request correctly

ISSUE: Mobile view doesn't collapse sidebar
FIX:
  1. Check browser is actually <768px wide
  2. Try clearing browser cache
  3. Check media query in CSS (bottom of style section)

ISSUE: Colors look wrong
FIX:
  1. Check CSS variables in :root section
  2. Verify fonts loaded from Google Fonts
  3. Check browser zoom level (Ctrl+0 to reset)
